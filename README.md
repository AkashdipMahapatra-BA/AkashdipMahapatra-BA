### 🚀 About Me: From Mechanical Design to Cloud Architecture
My engineering journey started in the physical world. With a background in Mechanical Engineering and a recognized passion for illustration (honored with a 2012 National Award from the President of India), I have always loved breaking down how complex systems fit together. 

Today, I apply that same spatial reasoning and structural mindset to distributed cloud systems, DevOps pipelines, and Site Reliability Engineering (SRE). I specialize in automating away toil, securing infrastructure, and ensuring high-availability enterprise environments run seamlessly.

---

### 🧠 The Engineering Mindset: Beyond the Defaults
I believe in understanding *why* a technology exists, not just *how* to use it. When I am not building infrastructure, I am deep-diving into the mechanics of next-generation tools:
* **Event Streaming at Scale:** Researching how massive platforms handle throughput bottlenecks beyond standard Kafka implementations (exploring architectural concepts like LinkedIn's advanced event-streaming iterations).
* **Version Control Under the Hood:** Analyzing Meta’s *Sapling* to understand how data structures and historical node snapshots operate differently from standard Git architecture.
* **System Design Visualization:** Leveraging my design background to mentally map and troubleshoot complex data flows, from S3 triggers down to individual Lambda execution environments. 

### ⚡ Current Focus & Continuous Learning
* **Orchestration:** Transitioning deeper into container orchestration by mastering **Kubernetes (K8s)**.
* **Backend Engineering:** Expanding my architectural scope by building Python-based web backends.
* **Algorithms:** Consistently sharpening my problem-solving efficiency and algorithmic thinking.

### 🛠️ Enterprise Cloud & DevOps Impact 
> *Note: Due to strict corporate NDAs and security policies, specific enterprise project code is maintained in private repositories. Below are high-level overviews of the architectural challenges I have solved in production environments.*

#### 1. Infrastructure & Deployment Automation
* **Zero-Touch QA Validation Engine:** Engineered an AWS-native Python automation tool using `boto3` to dynamically discover cloud resources (Lambdas, API Gateways, SNS) at runtime. Integrated with CloudWatch and Datadog APIs to automate deployment health checks, reducing manual sign-off time from ~30 minutes to under 3 minutes while enforcing a zero-error baseline.
* **CI/CD Pipeline Stabilization:** Investigated and resolved critical EventBridge-to-Lambda test execution failures by implementing robust pytest fixtures and improving containerized ETL environments. 

#### 2. Cloud FinOps & Observability
* **Cost Anomaly Root Cause Analysis:** Triaged runaway AWS billing alerts by cross-referencing CloudWatch metrics with CloudTrail logs. Identified and severed an infinite synchronous retry loop within an event-streaming architecture, securing cloud budgets and proposing permanent architectural safeguards.
* **Proactive System Monitoring:** Managed sophisticated Datadog monitors across multiple environments, tracking compute capacity, broker health, and cluster utilization to preemptively catch and resolve pipeline anomalies.

#### 3. Application Security & Cloud Hardening (DevSecOps)
* **IAM Least-Privilege Enforcement:** Authored and deployed Terraform configurations to systematically re-route IAM permissions and manage VPC endpoints across multi-environment API Gateways, ensuring seamless consumer transitions without service interruption.
* **Vulnerability Remediation:** Performed comprehensive static code analysis on a monolithic Python application. Successfully patched over 20 critical OWASP vulnerabilities (including SQLi, SSRF, and IDOR) and optimized Docker multi-stage builds to reduce container image sizes by 42%.
